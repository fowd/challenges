Final Project Requirements

Content

Your final site should have the following pages:

  • Home page (index.html): This page may have a brief (artist/mission) statement, artwork/work, or other introductory material
  • Contact page (contact.php): You will have a contact form you will develop in the next module, allowing visitors to the site to send you an email ... you will also have a thank you page, after the form is submitted
  • A page (or pages) with your main material ... this could be a portfolio gallery, your ministry, or product; each page should be named accordingly (i.e. gallery.html, ministry.html, widgets.html). You may have to create multiple galleries, if you want to showcase specific areas. You could also use Javascript and create "tabs"
  • A page that talks more about you (about.html) ... If you don't have a great deal to say, make this your resume page -- talk about your qualifications (your skill set and education). If you need, place your artist statement on this page too.

Single Page layout

You could compress all the content into one single HTML document. However, they do become extremely long and you must remember to keep the navigation where people can always get to it (fix the position or repeat the navigation in each section). Once again, Tabbed layouts could be used.

Coding Requirements

CSS

Reset

You must have at least a basic reset on your main css document (this is the CSS document that is listed first in your order)

* {margin: 0; padding: 0;}

Organize

Make sure you organize your CSS to the best of your ability. Try to group rules together that deal with:

  • Layout
  • Typography
  • Lists
  • Anchors
  • Misc

Also, you could divide up your CSS that deals with the front page and interior page layouts and typography

Make sure all CSS documents that you create are in a folder called "css" (all lowercase).

Validate

Make sure you validate your code for any major errors. Remember, try to have the validator check for CSS3 and avoid using too many vendor prefix rules (e.g. -webkit- and -moz-)

HTML

Validation

Make sure you validate your code to catch any errors!! Any errors will result in major point deductions .

Structure

Make sure you're not using too many divs and that you're using the id and class attributes (and correctly -- ids can only be used one time per page!)

Semantics

Is your use of header tags appropriate? Make sure you don't skip and there should only be one h1 tag per page. If you use an image for the title of your page, this is the best solution :

<h1>
  <a href="index.html">
    <img src="images/sitetitle.jpg" alt="Site Name Here" title="return to site name homepage">
  </a>
</h1>

JavaScript

Make sure you organize your JavaScript and any plugins (gallery, slideshow, tags, etc.) appropriately. While some JavaScript (like the jQuery framework code) will be in a folder title "js", some will be in it's parent plugin folder (fancyBox gallery, etc.).

Make sure you see me well before the final submission deadline, if you have questions or concerns!

Final Thoughts

Make sure you test your site thoroughly!

  • How does the site look in (at least) Chrome/Safari and Firefox?
  • Does the form work?
  • Does the gallery work?
  • Can you have one of your friends go through your site and not be confused?
  • Again, don't forget to validate
